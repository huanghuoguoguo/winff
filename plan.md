# WinFF 优化规划

> 定位：轻量级家居局域网文件共享服务
> 安全说明：不考虑认证机制，信任家庭局域网环境

---

## 📋 功能清单

### P0 - 高优先级（实用性强，实现简单）

#### 1. 拖拽上传 ✅
- [x] 支持拖拽文件到窗口任意位置上传
- [x] 拖拽时显示视觉反馈（高亮边框/遮罩）
- [ ] 支持拖拽文件夹（需要兼容浏览器支持）

**实现要点：**
- 在 `App.jsx` 添加全局 dragover/drop 事件监听
- 复用现有的 `UploadModal` 组件逻辑
- 使用 `DataTransfer.items` 处理文件夹

**实现状态：** 已完成（2026-03-29）

---

#### 2. 文件搜索功能 ✅
- [x] 在当前目录搜索文件/文件夹
- [x] 支持文件名模糊匹配
- [ ] 搜索结果高亮关键词
- [ ] 支持递归搜索子目录（可选开关）

**实现要点：**
- 前端输入框本地过滤（小目录）或后端 API 搜索（大目录/递归）
- 在 `FileList` 组件顶部添加搜索框
- 考虑防抖优化

**实现状态：** 已完成（2026-03-29）

---

#### 3. 上传进度优化 ✅
- [x] 显示当前上传文件的进度条
- [x] 显示上传速度/剩余时间估算
- [x] 支持取消正在进行的上传
- [ ] 上传完成后显示成功/失败列表

**实现要点：**
- 修改 `UploadModal.jsx` 增强进度展示
- XHR 的 `onprogress` 已可用，需增强 UI
- 使用 `AbortController` 支持取消

**实现状态：** 已完成（2026-03-29）

---

#### 4. 大目录分页/虚拟滚动 ✅
- [x] 超过阈值（如 500 个文件）自动启用虚拟滚动
- [x] 或实现分页加载（每页 100 条）
- [x] 保持滚动位置记忆

**实现要点：**
- 使用 `react-window` 或手动实现虚拟列表
- 后端 API 支持 `?limit=&offset=` 参数

**实现状态：** 已完成（2026-03-29）- 采用滚动加载方式

---

### P1 - 中优先级（提升体验）

#### 5. 文件/文件夹批量操作 ✅
- [x] 多选模式（长按/复选框）
- [x] 批量下载（打包 ZIP）
- [x] 批量删除
- [ ] 批量移动/复制

**实现要点：**
- 前端添加选择状态管理
- 后端添加 `/api/batch` 相关接口
- ZIP 打包使用 `archiver` 库

**实现状态：** 已完成（2026-03-29）- 批量下载临时采用逐个下载方式，批量删除待后端 API

---

#### 6. 图片缩略图缓存
- [ ] 服务端生成缩略图缓存
- [ ] 列表页优先加载缩略图
- [ ] 预览时加载原图
- [ ] 缓存清理机制

**实现要点：**
- 上传时自动生成 `.thumb/` 目录存放缩略图
- 添加 `/api/thumbnail?path=` 接口
- 使用 `sharp` 或 `jimp` 处理图片

---

#### 7. 文件夹上传支持
- [ ] 拖拽文件夹上传
- [ ] 保持目录结构
- [ ] 显示上传文件树

**实现要点：**
- `<input type="file" webkitdirectory>` 支持
- 解析 `File.webkitRelativePath` 保持结构
- 后端需支持创建子目录

---

#### 8. 断点续传
- [ ] 大文件上传支持暂停/恢复
- [ ] 服务端记录已上传分片
- [ ] 前端本地存储上传状态

**实现要点：**
- 文件分片上传（每片 5MB）
- 后端记录 `.upload_progress/` 状态
- 恢复时校验已上传分片

---

### P2 - 低优先级（锦上添花）

#### 9. 复制链接功能
- [ ] 一键复制文件访问链接
- [ ] 支持生成二维码分享
- [ ] 复制链接到剪贴板

**实现要点：**
- 使用 `navigator.clipboard.writeText`
- 复用现有的二维码组件

---

#### 10. 最近文件/历史记录
- [ ] 记录最近访问的 20 个文件
- [ ] 记录最近浏览的目录
- [ ] 快速返回上次浏览位置

**实现要点：**
- localStorage 存储历史记录
- 侧边栏添加"最近访问"入口

---

#### 11. 收藏夹功能
- [ ] 收藏常用文件/文件夹
- [ ] 侧边栏快速访问收藏
- [ ] 收藏数据持久化

**实现要点：**
- 收藏列表存入 `config.json` 或单独 JSON 文件
- 前端添加收藏管理 UI

---

#### 12. 服务端日志系统
- [ ] 日志文件输出 (`logs/access.log`, `logs/error.log`)
- [ ] 日志轮转（按天切割）
- [ ] 可配置的日志级别

**实现要点：**
- 使用 `winston` 或 `pino` 日志库
- 替换所有 `console.log`

---

#### 13. 配置文件热加载
- [ ] 监听 `config.json` 变化
- [ ] 自动重新加载共享目录配置
- [ ] 管理 UI 提供配置界面（可选）

**实现要点：**
- `fs.watchFile` 监听文件变化
- 添加管理页面 `/admin`（可选）
